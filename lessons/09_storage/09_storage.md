# 9. Хранение данных

## 1. Типы хранилищ во Flutter

| Тип | Инструменты | Когда использовать |
|-----|------------|-------------------|
| Простые пары ключ-значение | `SharedPreferences`, `flutter_secure_storage` | Настройки, токены, флаги |
| Реляционная БД (SQL) | `sqflite`, `Drift` | Структурированные данные со связями |
| NoSQL / объектная БД | `Hive`, `Isar` | Быстрое хранение объектов без SQL |
| Файловая система | `path_provider`, `dart:io` | Кеш изображений, документы, экспорт |

---

## 2. Когда что выбирать

```
Настройки пользователя (тема, язык)
  → SharedPreferences

Авторизационные токены
  → flutter_secure_storage (зашифровано)

Список задач, заметки, каталог товаров
  → Hive / Isar (простота) или Drift / sqflite (запросы)

Данные с отношениями (пользователи + заказы + товары)
  → Drift (рекомендуется) или sqflite

Временный кеш данных с API
  → Hive / Isar (быстро) или in-memory (Map)

Экспорт PDF, скачанные файлы
  → path_provider + dart:io
```

---

## 3. Архитектура хранилища

```
UI (Widget)
     ↓
Repository (интерфейс)
     ↓
LocalDataSource    ←→   RemoteDataSource
(SharedPrefs/Hive)       (Dio/http)
```

Локальный источник данных — зеркало удалённого. При offline — читаем из кеша.

---

## 4. Рекомендации по выбору

1. **SharedPreferences** — только для примитивных типов (String, int, bool, double, List\<String\>).
2. **flutter_secure_storage** — для любых данных, требующих шифрования (токены, пароли).
3. **Hive/Isar** — если нужна быстрая NoSQL БД без SQL-запросов.
4. **Drift** — если нужны сложные JOIN, транзакции, миграции.
5. **Не храни данные в виджете** — только в Repository / LocalDataSource.
