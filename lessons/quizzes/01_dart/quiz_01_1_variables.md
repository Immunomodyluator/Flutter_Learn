### Вопрос 1 🟢

Какой тип выведет Dart для переменной `var x = 42;`?

- A) `dynamic`
- B) `Object`
- C) `int`
- D) `num`

<details>
<summary>Ответ</summary>

**C) `int`**

Dart использует вывод типов (type inference): `var` при инициализации целым числом автоматически получает тип `int`. Это статический тип — он зафиксирован на этапе компиляции.

</details>

### Вопрос 2 🟢

Какое ключевое слово используется для объявления константы, вычисляемой **во время компиляции**?

- A) `final`
- B) `const`
- C) `static`
- D) `readonly`

<details>
<summary>Ответ</summary>

**B) `const`**

`const` — compile-time константа. `final` позволяет присвоить значение один раз, но оно может вычисляться во время выполнения (`final now = DateTime.now()`). `const` требует полностью известного значения на этапе компиляции.

</details>

### Вопрос 3 🟢

Что выведет следующий код?

```dart
final List<int> nums = [1, 2, 3];
nums.add(4);
print(nums.length);
```

- A) Ошибка компиляции
- B) Ошибка выполнения
- C) `4`
- D) `3`

<details>
<summary>Ответ</summary>

**C) `4`**

`final` запрещает **переприсвоение** переменной, но не изменение объекта, на который она указывает. Список изменяемый, поэтому `add()` работает. Для запрета изменений нужен `const List<int>`.

</details>

### Вопрос 4 🟡

Какой тип следует использовать, когда переменная может хранить и `int`, и `double`?

- A) `dynamic`
- B) `Object`
- C) `num`
- D) `var`

<details>
<summary>Ответ</summary>

**C) `num`**

`num` — общий суперкласс для `int` и `double`. Это безопасный статически типизированный вариант. `dynamic` отключает статическую проверку типов. `Object` слишком широк — нет арифметических операций.

</details>

### Вопрос 5 🟡

Что происходит при присвоении `late` переменной до её инициализации?

- A) Возвращается `null`
- B) Бросается `LateInitializationError` в runtime
- C) Ошибка компиляции
- D) Возвращается значение по умолчанию для типа

<details>
<summary>Ответ</summary>

**B) Бросается `LateInitializationError` в runtime**

`late` откладывает обязательную инициализацию до момента первого обращения. Если к переменной обратиться до инициализации — будет брошено `LateInitializationError`. Компилятор этого не проверяет.

</details>

### Вопрос 6 🟡

Чем `dynamic` отличается от `Object?`?

- A) Никакой разницы
- B) `dynamic` отключает статическую проверку, `Object?` сохраняет её
- C) `Object?` быстрее в runtime
- D) `dynamic` не допускает `null`, `Object?` допускает

<details>
<summary>Ответ</summary>

**B) `dynamic` отключает статическую проверку, `Object?` сохраняет её**

С `dynamic` компилятор разрешает любые вызовы методов без проверки. С `Object?` нужно явно приводить тип или проверять перед вызовом методов, не входящих в интерфейс `Object?`.

</details>

### Вопрос 7 🟡

Какой будет результат?

```dart
const a = [1, 2, 3];
a.add(4);
```

- A) `[1, 2, 3, 4]`
- B) Ошибка компиляции
- C) `UnsupportedError` во время выполнения
- D) `[1, 2, 3]` без изменений

<details>
<summary>Ответ</summary>

**C) `UnsupportedError` во время выполнения**

`const` список неизменяем (immutable). При попытке вызвать `add()` бросается `UnsupportedError: Cannot add to an unmodifiable list`. Компилятор этого не поймает за исключением явного `const [].add(...)`.

</details>

### Вопрос 8 🔴

Какой тип имеет литерал `1` в выражении `double x = 1;`?

- A) `int`, затем неявно конвертируется в `double`
- B) `double` сразу
- C) `num`
- D) Ошибка компиляции — нужно писать `1.0`

<details>
<summary>Ответ</summary>

**B) `double` сразу**

В Dart целочисленный литерал в контексте `double` автоматически трактуется как `double`-литерал. Это особое правило вывода типов: `double x = 1` эквивалентно `double x = 1.0`. Никакого неявного приведения типов нет — компилятор понимает литерал как `double`.

</details>

### Вопрос 9 🔴

Что выведет код?

```dart
var x = 0;
x += 1.5;
print(x);
```

- A) `1.5`
- B) `1`
- C) Ошибка компиляции
- D) `2`

<details>
<summary>Ответ</summary>

**C) Ошибка компиляции**

`var x = 0` выводит тип `int`. Оператор `+=` с `double` (1.5) несовместим с `int`, поскольку `int + double` возвращает `double`, а присвоить `double` переменной типа `int` нельзя. Dart это отловит статически.

</details>

### Вопрос 10 🔴

Для чего нужен тип `Never` в Dart?

- A) Обозначает переменную, которая никогда не инициализируется
- B) Возвращаемый тип функции, которая никогда не завершается нормально
- C) Синоним `void`
- D) Тип для пустых коллекций

<details>
<summary>Ответ</summary>

**B) Возвращаемый тип функции, которая никогда не завершается нормально**

`Never` — тип-дно (bottom type). Функция с типом `Never` либо бросает исключение, либо уходит в бесконечный цикл. Анализатор flow analysis использует это: после вызова `Never`-функции код считается недостижимым. Пример: `throw` — выражение типа `Never`.

</details>
