### Вопрос 1 🟢
Какая коллекция хранит пары ключ-значение в Dart?

- A) `List`
- B) `Set`
- C) `Map`
- D) `Queue`

<details>
<summary>Ответ</summary>

**C) `Map`**

`Map<K, V>` — коллекция пар ключ-значение. `List` — упорядоченный список с индексами, `Set` — неупорядоченное множество уникальных элементов.

</details>

### Вопрос 2 🟢
Как создать неизменяемый список из 3 элементов?

- A) `List.immutable([1, 2, 3])`
- B) `const [1, 2, 3]`
- C) `final [1, 2, 3]`
- D) `List.fixed(3)`

<details>
<summary>Ответ</summary>

**B) `const [1, 2, 3]`**

`const` создаёт compile-time константный список, который нельзя изменить. `final` только запрещает переприсвоение переменной, но список остаётся изменяемым.

</details>

### Вопрос 3 🟢
Что гарантирует тип `Set` в Dart?

- A) Упорядоченность элементов
- B) Уникальность элементов
- C) Быстрый доступ по индексу
- D) Хранение пар ключ-значение

<details>
<summary>Ответ</summary>

**B) Уникальность элементов**

`Set` не допускает дубликатов. При добавлении уже существующего элемента `Set` просто игнорирует его. Порядок элементов в `HashSet` не гарантирован (существует `LinkedHashSet` с сохранением порядка вставки).

</details>

### Вопрос 4 🟡
Что вернёт `[1, 2, 3].map((e) => e * 2)`?

- A) `[2, 4, 6]`
- B) `(2, 4, 6)`
- C) `Iterable<int>`
- D) `List<int>`

<details>
<summary>Ответ</summary>

**C) `Iterable<int>`**

`map()` возвращает ленивый `Iterable`, а не `List`. Для получения `List` нужно вызвать `.toList()`: `[1, 2, 3].map((e) => e * 2).toList()`.

</details>

### Вопрос 5 🟡
Что произойдёт при обращении `map['key']`, если ключ не существует?

- A) Бросается `KeyError`
- B) Бросается `NoSuchMethodError`
- C) Возвращается `null`
- D) Возвращается пустая строка

<details>
<summary>Ответ</summary>

**C) Возвращается `null`**

Оператор `[]` у `Map` возвращает `V?` (nullable). Если ключ не найден — возвращается `null`. Поэтому тип результата `map['key']` всегда nullable, даже если `V` — non-nullable тип.

</details>

### Вопрос 6 🟡
Какой метод удалит из списка все элементы, удовлетворяющие условию?

- A) `list.remove((e) => e > 5)`
- B) `list.removeWhere((e) => e > 5)`
- C) `list.filter((e) => e > 5)`
- D) `list.delete((e) => e > 5)`

<details>
<summary>Ответ</summary>

**B) `list.removeWhere((e) => e > 5)`**

`removeWhere` изменяет список на месте, удаляя все элементы, для которых предикат возвращает `true`. `remove()` удаляет только первое вхождение конкретного значения, а не по условию.

</details>

### Вопрос 7 🟡
Что делает spread оператор `...` в литерале списка?

- A) Копирует список по ссылке
- B) Разворачивает итерируемый объект в элементы списка
- C) Создаёт глубокую копию
- D) Объединяет типы

<details>
<summary>Ответ</summary>

**B) Разворачивает итерируемый объект в элементы списка**

`[...list1, ...list2]` — создаёт новый список, объединяя элементы обоих. `...?` — null-safe версия, игнорирует `null`. Это синтаксический сахар, компилируется в серию `addAll`.

</details>

### Вопрос 8 🔴
Какова сложность поиска элемента в `HashSet`?

- A) O(n)
- B) O(log n)
- C) O(1) амортизированно
- D) O(n log n)

<details>
<summary>Ответ</summary>

**C) O(1) амортизированно**

`HashSet` использует хеш-таблицу. Поиск, вставка и удаление — O(1) в среднем случае. В наихудшем случае (много коллизий) — O(n), но на практике это крайне редко.

</details>

### Вопрос 9 🔴
Что выведет этот код?
```dart
final map = {'a': 1, 'b': 2};
map.update('c', (v) => v + 1, ifAbsent: () => 0);
print(map);
```

- A) `{a: 1, b: 2}`
- B) `{a: 1, b: 2, c: 1}`
- C) `{a: 1, b: 2, c: 0}`
- D) Бросается `StateError`

<details>
<summary>Ответ</summary>

**C) `{a: 1, b: 2, c: 0}`**

`update` вызывает первую функцию для существующего ключа, а `ifAbsent` — для отсутствующего. Ключ `'c'` не существует, поэтому вызывается `ifAbsent: () => 0`, и добавляется запись `'c': 0`.

</details>

### Вопрос 10 🔴
В чём разница между `List.filled` и `List.generate`?

- A) Никакой разницы
- B) `filled` создаёт список с одним значением для всех элементов; `generate` вычисляет каждый элемент через функцию от индекса
- C) `filled` изменяемый, `generate` — нет
- D) `generate` не принимает тип

<details>
<summary>Ответ</summary>

**B) `filled` создаёт список с одним значением для всех элементов; `generate` вычисляет каждый элемент через функцию от индекса**

`List.filled(5, 0)` → `[0, 0, 0, 0, 0]`. Все элементы — одна и та же ссылка на объект!  
`List.generate(5, (i) => i * 2)` → `[0, 2, 4, 6, 8]`. Каждый элемент — результат функции. Для объектов лучше использовать `generate`, чтобы избежать общей ссылки.

</details>
