### Вопрос 1 🟢

Как сделать параметр функции необязательным и именованным?

- A) `void foo(int x)`
- B) `void foo([int x])`
- C) `void foo({int? x})`
- D) `void foo(optional int x)`

<details>
<summary>Ответ</summary>

**C) `void foo({int? x})`**

Фигурные скобки `{}` создают именованные параметры. Для необязательного именованного параметра тип должен быть nullable (`int?`) или иметь значение по умолчанию. Квадратные скобки `[]` создают позиционные необязательные параметры.

</details>

### Вопрос 2 🟢

Что такое функция первого класса (first-class function) в Dart?

- A) Функция с ключевым словом `first`
- B) Функция, которую можно передавать как значение, хранить в переменной и возвращать
- C) Только глобальные функции
- D) Функции с типом `void`

<details>
<summary>Ответ</summary>

**B) Функция, которую можно передавать как значение, хранить в переменной и возвращать**

В Dart функции — объекты класса `Function`. Их можно передавать в другие функции как аргументы, хранить в переменных и возвращать из функций. Это основа функционального программирования.

</details>

### Вопрос 3 🟢

Что делает оператор `=>` в объявлении функции?

- A) Возвращает `Future`
- B) Краткая запись для функции с одним выражением (`return`)
- C) Создаёт стрелочный тип
- D) Определяет тип возвращаемого значения

<details>
<summary>Ответ</summary>

**B) Краткая запись для функции с одним выражением (`return`)**

`int double(int x) => x * 2;` — эквивалентно `int double(int x) { return x * 2; }`. Работает только с одним выражением, не с блоком операторов.

</details>

### Вопрос 4 🟡

Что выведет этот код?

```dart
void main() {
  var fns = <Function>[];
  for (var i = 0; i < 3; i++) {
    fns.add(() => print(i));
  }
  fns.forEach((f) => f());
}
```

- A) `0`, `1`, `2`
- B) `2`, `2`, `2`
- C) `3`, `3`, `3`
- D) Ошибка компиляции

<details>
<summary>Ответ</summary>

**C) `3`, `3`, `3`**

Замыкания в Dart захватывают переменную **по ссылке**, а не по значению. К моменту вызова функций цикл завершился, и `i` равно `3`. Это классическая ловушка замыканий в циклах. Для фиксации значения нужно: `var j = i; fns.add(() => print(j));`

</details>

### Вопрос 5 🟡

Что такое `typedef` в Dart?

- A) Псевдоним для любого типа, включая функциональные
- B) Только для функциональных типов
- C) Синоним `abstract class`
- D) Псевдоним только для примитивных типов

<details>
<summary>Ответ</summary>

**A) Псевдоним для любого типа, включая функциональные**

С Dart 2.13+ `typedef` работает с любыми типами: `typedef IntList = List<int>;`. Исторически использовался только для функций: `typedef Predicate = bool Function(int)`. Современный `typedef` мощнее.

</details>

### Вопрос 6 🟡

Метод `call()` в классе позволяет:

- A) Вызывать приватные методы снаружи
- B) Использовать экземпляр класса как функцию
- C) Создавать конструктор без имени
- D) Вызывать статические методы через экземпляр

<details>
<summary>Ответ</summary>

**B) Использовать экземпляр класса как функцию**

Если класс определяет метод `call()`, его экземпляры можно вызывать как функции: `obj(args)`. Это делает класс callable и позволяет использовать его там, где ожидается `Function`.

</details>

### Вопрос 7 🟡

Чем отличается обязательный именованный параметр от необязательного?

- A) Синтаксически ничем
- B) Обязательный помечен `required`, необязательный имеет `?` или default value
- C) Обязательный использует `[]`, необязательный `{}`
- D) Обязательный всегда первый в списке параметров

<details>
<summary>Ответ</summary>

**B) Обязательный помечен `required`, необязательный имеет `?` или default value**

`void foo({required int x, int y = 0, String? z})` — `x` обязателен, `y` необязателен со значением по умолчанию, `z` необязателен и может быть null. Ключевое слово `required` проверяется компилятором.

</details>

### Вопрос 8 🔴

Что такое tear-off функции в Dart?

- A) Функция, которая освобождает память
- B) Ссылка на метод объекта или конструктор без явного вызова
- C) Функция-деструктор
- D) Функция с автоматическим `try-catch`

<details>
<summary>Ответ</summary>

**B) Ссылка на метод объекта или конструктор без явного вызова**

`list.forEach(print)` — `print` это tear-off функции `print()`. Вместо `(x) => print(x)`. Аналогично: `['a','b'].map(String.toUpperCase)` или использование конструктора `MyClass.new` как функции. Tear-offs эффективнее лямбд, так как не создают новый объект при каждом вызове.

</details>

### Вопрос 9 🔴

Что выведет код?

```dart
Function makeAdder(int x) => (int y) => x + y;

void main() {
  final add5 = makeAdder(5);
  final add10 = makeAdder(10);
  print(add5(3));
  print(add10(3));
}
```

- A) `8` и `8`
- B) `8` и `13`
- C) `13` и `8`
- D) Ошибка: нельзя возвращать функцию

<details>
<summary>Ответ</summary>

**B) `8` и `13`**

`makeAdder` — функция высшего порядка (currying). Каждый вызов создаёт новое замыкание, захватывающее свой `x`. `add5` захватывает `x=5`, `add10` — `x=10`. Они независимы: `5+3=8`, `10+3=13`.

</details>

### Вопрос 10 🔴

Каков тип функции `bool Function(int, {String label})`?

- A) Нельзя создать такой тип
- B) Функция с обязательным позиционным `int` и обязательным именованным `String`
- C) Функция с обязательным позиционным `int` и необязательным именованным `String?`
- D) Тип недопустим — именованные параметры не включаются в `Function` тип

<details>
<summary>Ответ</summary>

**B) Функция с обязательным позиционным `int` и обязательным именованным `String`**

В Dart в типе функции именованные параметры без `?` и без `=` являются **обязательными** (`required`). Тип `bool Function(int, {String label})` означает: `label` обязателен. Если нужен необязательный — `bool Function(int, {String? label})`.

</details>
