### Вопрос 1 🟢

Что такое `abstract class` в Dart?

- A) Класс без конструктора
- B) Класс, от которого нельзя создать экземпляр напрямую; служит базовым типом
- C) Класс только со статическими методами
- D) Класс без полей

<details>
<summary>Ответ</summary>

**B) Класс, от которого нельзя создать экземпляр напрямую; служит базовым типом**

`abstract class` можно наследовать (`extends`) или реализовывать (`implements`). Попытка `AbstractClass()` — ошибка компиляции.

</details>

### Вопрос 2 🟢

Как в Dart обозначить приватный метод или поле?

- A) Ключевым словом `private`
- B) Префиксом `_` (нижним подчёркиванием)
- C) Модификатором `protected`
- D) Вложением в `private { }`

<details>
<summary>Ответ</summary>

**B) Префиксом `_` (нижним подчёркиванием)**

В Dart приватность — на уровне **библиотеки** (файла), не класса. Идентификатор с `_` недоступен за пределами файла, в котором определён. Ключевых слов `private`/`protected` в Dart нет.

</details>

### Вопрос 3 🟢

Какое ключевое слово используется для реализации интерфейса в Dart?

- A) `extends`
- B) `implements`
- C) `with`
- D) `interface`

<details>
<summary>Ответ</summary>

**B) `implements`**

`implements` требует реализовать **все** методы и свойства интерфейса. В Dart любой класс автоматически является интерфейсом. `extends` — наследование с переиспользованием реализации. `with` — миксины.

</details>

### Вопрос 4 🟡

Что такое mixin в Dart?

- A) Анонимный класс
- B) Способ переиспользовать код в нескольких иерархиях классов без наследования
- C) Синоним интерфейса
- D) Класс с фабричным конструктором

<details>
<summary>Ответ</summary>

**B) Способ переиспользовать код в нескольких иерархиях классов без наследования**

`mixin` позволяет «подмешать» реализацию методов в класс через `with`. Dart не поддерживает множественное наследование, но `with` позволяет использовать несколько миксинов одновременно: `class Foo extends Bar with MixA, MixB`.

</details>

### Вопрос 5 🟡

Что выведет код?

```dart
class A {
  String hello() => 'A';
}
class B extends A {
  @override
  String hello() => 'B';
}
void main() {
  A obj = B();
  print(obj.hello());
}
```

- A) `'A'`
- B) `'B'`
- C) Ошибка компиляции
- D) `'AB'`

<details>
<summary>Ответ</summary>

**B) `'B'`**

Dart использует **динамическую диспетчеризацию** (полиморфизм). Переменная объявлена как `A`, но реальный объект — `B`. Вызывается метод реального типа объекта, т.е. `B.hello()`. Это и есть полиморфизм.

</details>

### Вопрос 6 🟡

Что делает `factory` конструктор в Dart?

- A) Всегда создаёт новый объект
- B) Может возвращать существующий экземпляр или экземпляр подтипа
- C) Создаёт только статические объекты
- D) Аналог `clone()` в Java

<details>
<summary>Ответ</summary>

**B) Может возвращать существующий экземпляр или экземпляр подтипа**

`factory` конструктор не обязан создавать новый объект. Используется для паттерна Singleton (`return _instance`), кэширования, или возврата подтипа. Он имеет доступ к `return`, в отличие от обычных конструкторов.

</details>

### Вопрос 7 🟡

Что такое `interface class` (Dart 3+)?

- A) Класс только с абстрактными методами
- B) Класс, который можно только реализовывать, но не наследовать
- C) Класс только для UI
- D) Синоним `abstract class`

<details>
<summary>Ответ</summary>

**B) Класс, который можно только реализовывать, но не наследовать**

Dart 3 ввёл модификаторы классов: `interface class` запрещает `extends` (только `implements`). `base class` — только `extends` (не `implements`). `final class` — ни то, ни другое вне библиотеки. `sealed class` — для exhaustive pattern matching.

</details>

### Вопрос 8 🔴

Каков порядок инициализации в Dart при `class B extends A`?

- A) Поля B → конструктор A → конструктор B
- B) Конструктор A → поля B → конструктор B
- C) Initializer list B → super() → тело конструктора B
- D) Тело конструктора B → super() → поля A

<details>
<summary>Ответ</summary>

**C) Initializer list B → super() → тело конструктора B**

Порядок инициализации в Dart: 1) Initializer list текущего класса (включая вызов `super()`), 2) Тело конструктора суперкласса, 3) Тело конструктора текущего класса. Поля с initializers инициализируются до входа в тело конструктора.

</details>

### Вопрос 9 🔴

Что такое `extension methods` в Dart?

- A) Методы, которые можно переопределить во всех подклассах
- B) Добавление методов к существующим типам без изменения их исходного кода
- C) Методы с расширенными правами доступа
- D) Методы для работы с файлами

<details>
<summary>Ответ</summary>

**B) Добавление методов к существующим типам без изменения их исходного кода**

```dart
extension StringUtils on String {
  bool get isPalindrome => this == split('').reversed.join();
}
// 'racecar'.isPalindrome → true
```

Extension методы — синтаксический сахар: компилятор превращает `obj.method()` в `ExtName.method(obj)`.

</details>

### Вопрос 10 🔴

В чём разница между `sealed class` и `abstract class` в Dart 3?

- A) Никакой разницы
- B) `sealed class` ограничивает наследников одной библиотекой, делая `switch` exhaustive
- C) `sealed class` нельзя использовать с `implements`
- D) `sealed class` не поддерживает поля

<details>
<summary>Ответ</summary>

**B) `sealed class` ограничивает наследников одной библиотекой, делая `switch` exhaustive**

`sealed class` в Dart 3 делает иерархию «закрытой»: все подтипы должны быть в той же библиотеке. Компилятор знает все возможные подтипы, поэтому `switch` по sealed типу проверяет покрытие всех случаев — пропущенный вариант — ошибка компиляции. Это основа паттерна Union Types.

</details>
