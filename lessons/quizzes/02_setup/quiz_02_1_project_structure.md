### Вопрос 1 🟢

В какой папке находится точка входа Flutter-приложения?

- A) `src/main.dart`
- B) `lib/main.dart`
- C) `app/main.dart`
- D) `flutter/main.dart`

<details>
<summary>Ответ</summary>

**B) `lib/main.dart`**

`lib/` — главная директория Dart-кода. `main.dart` содержит функцию `main()` с вызовом `runApp()`. Все публичные пакеты и библиотеки также экспортируются из `lib/`.

</details>

### Вопрос 2 🟢

Где регистрируются зависимости Flutter-проекта?

- A) `package.json`
- B) `build.gradle`
- C) `pubspec.yaml`
- D) `flutter.config`

<details>
<summary>Ответ</summary>

**C) `pubspec.yaml`**

`pubspec.yaml` — манифест Flutter/Dart проекта. Содержит: имя и версию, зависимости (`dependencies`), dev-зависимости, ресурсы (`flutter.assets`), шрифты и минимальные версии SDK.

</details>

### Вопрос 3 🟢

Для чего нужна папка `test/` в Flutter-проекте?

- A) Хранение тестовых данных (JSON, изображения)
- B) Unit и widget тесты
- C) Интеграционные тесты
- D) Конфигурация тестовой среды

<details>
<summary>Ответ</summary>

**B) Unit и widget тесты**

`test/` содержит unit и widget тесты. Интеграционные тесты размещаются в `integration_test/` (отдельный пакет). E2E тесты с `patrol` — тоже в `integration_test/`.

</details>

### Вопрос 4 🟡

Что такое `.dart_tool/` в корне проекта?

- A) Пользовательские Dart инструменты
- B) Служебная директория: кэш `pub`, информация об окружении — не нужно коммитить
- C) Директория для Dart скриптов
- D) Локальные настройки анализатора

<details>
<summary>Ответ</summary>

**B) Служебная директория: кэш `pub`, информация об окружении — не нужно коммитить**

`.dart_tool/` автоматически генерируется `flutter pub get`. Содержит `package_config.json` (маппинг пакетов), кэш генераторов кода. Должна быть в `.gitignore`.

</details>

### Вопрос 5 🟡

Что хранится в `pubspec.lock`?

- A) Зашифрованные ключи пакетов
- B) Зафиксированные точные версии всех зависимостей (включая транзитивные)
- C) Кэш загруженных пакетов
- D) Список запрещённых версий пакетов

<details>
<summary>Ответ</summary>

**B) Зафиксированные точные версии всех зависимостей (включая транзитивные)**

`pubspec.lock` гарантирует воспроизводимость сборки: все разработчики и CI получат одинаковые версии. Для приложений — коммитить в Git. Для пакетов — обычно не коммитят (в `.gitignore`), чтобы не ограничивать пользователей.

</details>

### Вопрос 6 🟡

Для чего нужна папка `android/` и `ios/` в корне Flutter проекта?

- A) Только для иконок приложения
- B) Нативный код платформы, манифесты, настройки сборки
- C) Тесты для конкретных платформ
- D) Автоматически удаляются при сборке

<details>
<summary>Ответ</summary>

**B) Нативный код платформы, манифесты, настройки сборки**

`android/` содержит `AndroidManifest.xml`, `build.gradle`, нативный Kotlin/Java код. `ios/` — `Info.plist`, `Podfile`, Swift/Objective-C код. Flutter plugins добавляют свой нативный код именно сюда через pub publishing.

</details>

### Вопрос 7 🟡

Где должны быть объявлены пути к картинкам в проекте, чтобы их можно было использовать через `Image.asset()`?

- A) Только в папке `lib/assets/`
- B) В секции `flutter.assets` в `pubspec.yaml`
- C) В файле `assets.dart`
- D) Автоматически — достаточно положить картинку в `assets/`

<details>
<summary>Ответ</summary>

**B) В секции `flutter.assets` в `pubspec.yaml`**

```yaml
flutter:
  assets:
    - assets/images/
    - assets/icons/logo.png
```

Без регистрации в `pubspec.yaml` файлы не будут включены в сборку и `Image.asset()` выбросит ошибку. Можно указать папку с `/` — включит все файлы в ней (не рекурсивно).

</details>

### Вопрос 8 🔴

В чём назначение `analysis_options.yaml`?

- A) Настройки производительности анализатора кода
- B) Конфигурация статического анализатора: правила линтера, включение/исключение файлов
- C) Настройки форматирования кода (отступы, длина строки)
- D) Список файлов для обфускации

<details>
<summary>Ответ</summary>

**B) Конфигурация статического анализатора: правила линтера, включение/исключение файлов**

```yaml
include: package:flutter_lints/flutter.yaml
linter:
  rules:
    prefer_const_constructors: true
    avoid_print: true
analyzer:
  exclude:
    - "**/*.g.dart"
```

Контролирует предупреждения, ошибки и стиль кода. `flutter_lints` — рекомендованный набор правил.

</details>

### Вопрос 9 🔴

Что происходит с папкой `build/` при выполнении `flutter clean`?

- A) Удаляются только временные файлы
- B) Удаляется вся папка `build/`, очищается `.dart_tool/` кэш
- C) Удаляются зависимости из `pubspec.lock`
- D) Сбрасываются настройки Xcode

<details>
<summary>Ответ</summary>

**B) Удаляется вся папка `build/`, очищается `.dart_tool/` кэш**

`flutter clean` удаляет `build/` и `.dart_tool/`. После этого нужно `flutter pub get` и пересборка. Используется при проблемах со сборкой, несоответствии версий, странных ошибках. Не удаляет `pubspec.lock` или установленные пакеты из `~/.pub-cache`.

</details>

### Вопрос 10 🔴

Что означает `^3.0.0` в секции `dependencies` в `pubspec.yaml`?

- A) Точно версия `3.0.0`
- B) Любая версия `>= 3.0.0` и `< 4.0.0`
- C) Любая версия `>= 3.0.0`
- D) Версия `3.x.x`, где x > 0

<details>
<summary>Ответ</summary>

**B) Любая версия `>= 3.0.0` и `< 4.0.0`**

`^` — caret constraint в semver. Допускает минорные и патч обновления, но не мажорные (breaking changes). `^3.0.0` → `3.0.0 <= v < 4.0.0`. `^0.3.0` → `0.3.0 <= v < 0.4.0` (особый случай для версий до 1.0).

</details>
