# Квиз: Hot Reload и Hot Restart

**Тема:** 02.3 — Hot Reload & Hot Restart  
**Вопросов:** 10 | 🟢 1–3 · 🟡 4–7 · 🔴 8–10

---

### Вопрос 1 🟢

Какая клавиша вызывает Hot Reload в терминале при запущенном `flutter run`?

- A) `r`
- B) `h`
- C) `R`
- D) `F5`

<details>
<summary>Ответ</summary>

**A) `r`**

`r` — Hot Reload (быстро, сохраняет состояние). `R` — Hot Restart (медленнее, сбрасывает состояние). `q` — выход. `p` — включить/выключить линии сетки (performance overlay). `h` — показать все горячие клавиши.

</details>

### Вопрос 2 🟢

Сохраняет ли Hot Reload состояние приложения?

- A) Никогда не сохраняет
- B) Да, состояние виджетов и данных сохраняется
- C) Только состояние StatelessWidget
- D) Только глобальные переменные

<details>
<summary>Ответ</summary>

**B) Да, состояние виджетов и данных сохраняется**

Hot Reload внедряет изменённый код без перезапуска. `State` объекты сохраняются, поля не сбрасываются. Это позволяет видеть изменения UI мгновенно, оставаясь на текущем экране с текущими данными.

</details>

### Вопрос 3 🟢

Когда нужно использовать Hot Restart вместо Hot Reload?

- A) При любом изменении кода
- B) При изменениях в `initState()`, глобальных переменных, нативном коде
- C) При изменении цвета кнопки
- D) При добавлении нового виджета

<details>
<summary>Ответ</summary>

**B) При изменениях в `initState()`, глобальных переменных, нативном коде**

Hot Reload **не** применяет: изменения в конструкторах, `initState()`, глобальных/статических полях, изменения enum и generics. Для их применения нужен Hot Restart (сброс Dart VM, сохранение нативного состояния) или полный перезапуск.

</details>

### Вопрос 4 🟡

Что происходит с Dart VM при Hot Reload?

- A) VM перезапускается полностью
- B) Изменённые библиотеки компилируются и их классы обновляются «на лету» в работающей VM
- C) VM создаёт параллельный процесс
- D) VM сохраняет снапшот состояния

<details>
<summary>Ответ</summary>

**B) Изменённые библиотеки компилируются и их классы обновляются «на лету» в работающей VM**

Flutter компилирует только изменённые библиотеки, затем отправляет новый байткод в работающую Dart VM. VM меняет определения классов без пересоздания экземпляров. Флаттер вызывает `reassemble()` → перестраивает дерево виджетов.

</details>

### Вопрос 5 🟡

Почему Hot Reload недоступен в release-сборке?

- A) Ограничение лицензии
- B) Release использует AOT компиляцию — обновление кода во время выполнения невозможно
- C) Hot Reload требует интернет-соединения
- D) Release сборка не поддерживает Dart VM

<details>
<summary>Ответ</summary>

**B) Release использует AOT компиляцию — обновление кода во время выполнения невозможно**

Debug режим использует JIT (Just-In-Time) компиляцию с поддержкой динамического обновления. Release режим компилирует в нативный код заранее (AOT — Ahead-Of-Time). AOT даёт максимальную производительность, но не допускает изменений во время выполнения.

</details>

### Вопрос 6 🟡

Что такое Stateful Hot Reload и в чём его ограничение при изменении типа поля `State`?

- A) Нет ограничений — всё обновляется
- B) Если тип поля изменился (например `int` → `String`) — Hot Reload не срабатывает, нужен Restart
- C) Можно изменить только nullable поля
- D) Ограничение только для `final` полей

<details>
<summary>Ответ</summary>

**B) Если тип поля изменился (например `int` → `String`) — Hot Reload не срабатывает, нужен Restart**

При Hot Reload Flutter пытается сохранить существующие `State` объекты. Если изменился тип поля — несоответствие типов в существующем объекте. Flutter покажет ошибку и предложит Hot Restart или выдаст предупреждение о невозможности обновления.

</details>

### Вопрос 7 🟡

Метод `reassemble()` в `State` вызывается при:

- A) Каждом `setState()`
- B) Hot Reload для корректного обновления виджетов
- C) Переходе между экранами
- D) Изменении темы приложения

<details>
<summary>Ответ</summary>

**B) Hot Reload для корректного обновления виджетов**

`reassemble()` вызывается Flutter при Hot Reload. По умолчанию вызывает `markNeedsBuild()`. Можно переопределить для специальной логики при Hot Reload (например, перечитать файл конфигурации). В production никогда не вызывается.

</details>

### Вопрос 8 🔴

Почему добавление нового элемента в enum требует Hot Restart?

- A) Enum нельзя изменять вообще
- B) Изменение структуры enum меняет бинарное представление класса, несовместимое с работающей VM
- C) Hot Reload не умеет работать с enum
- D) Enum автоматически перезапускают приложение

<details>
<summary>Ответ</summary>

**B) Изменение структуры enum меняет бинарное представление класса, несовместимое с работающей VM**

Hot Reload ограничен: нельзя добавлять новые поля в класс (только изменять тела методов), нельзя менять суперклассы, нельзя добавлять константы enum. Такие изменения требуют полного переинициализации объектов — нужен Restart.

</details>

### Вопрос 9 🔴

В каком режиме оптимально профилировать производительность Flutter-приложения?

- A) Debug
- B) Release
- C) Profile
- D) JIT mode

<details>
<summary>Ответ</summary>

**C) Profile**

Profile режим (`flutter run --profile`) — скомпилирован как release (AOT, нет отладочных проверок), но с включёнными observatory и DevTools. Даёт реальные цифры производительности без overhead debug-режима. Release режим отключает все инструменты профилирования.

</details>

### Вопрос 10 🔴

Что такое `flutter run -d web-server` и когда это полезно?

- A) Запускает бэкенд на локальном сервере
- B) Запускает Flutter Web приложение с HTML/JS сервером для тестирования в браузере
- C) Синхронизирует с удалённым устройством
- D) Запускает приложение на всех устройствах одновременно

<details>
<summary>Ответ</summary>

**B) Запускает Flutter Web приложение с HTML/JS сервером для тестирования в браузере**

`flutter run -d web-server --web-port=8080` запускает dev-сервер. `--web-renderer html` или `canvaskit` — выбор рендерера. Hot Reload работает и для Web: изменения применяются через websocket соединение с браузером. Полезно при разработке web-версии без запуска мобильных эмуляторов.

</details>
