# 08.2 Dio — продвинутый HTTP-клиент

## 1. Суть

`Dio` — самый популярный HTTP-клиент для Flutter/Dart. Превосходит `http` по функциональности: интерцепторы, ретраи, отмена запросов, FormData, загрузка файлов с прогрессом, глобальная обработка ошибок.

```yaml
# pubspec.yaml
dependencies:
  dio: ^5.4.0
```

---

## 2. Как используется во Flutter

Dio создаётся один раз (синглтон) через DI и передаётся в Repository. Настраивается через `BaseOptions` и `Interceptor`.

```dart
import 'package:dio/dio.dart';
```

---

## 3. Основной синтаксис

```dart
// Создание с настройками
final dio = Dio(
  BaseOptions(
    baseUrl: 'https://api.example.com',
    connectTimeout: const Duration(seconds: 10),
    receiveTimeout: const Duration(seconds: 15),
    headers: {'Accept': 'application/json'},
  ),
);

// GET
final response = await dio.get<Map<String, dynamic>>('/users/1');

// GET с параметрами
final response = await dio.get<List<dynamic>>(
  '/products',
  queryParameters: {'page': 1, 'limit': 20, 'sort': 'price'},
);

// POST с JSON телом
final response = await dio.post<Map<String, dynamic>>(
  '/users',
  data: {'name': 'Ivan', 'email': 'ivan@example.com'},
);

// PUT / PATCH / DELETE
await dio.put('/users/1', data: updatedData);
await dio.patch('/users/1', data: partialData);
await dio.delete('/users/1');

// Доступ к данным
response.data        // динамически типизированные данные
response.statusCode  // int
response.headers     // Headers
```

## 4. Интерцепторы

```dart
// Интерцептор авторизации (добавляет Bearer токен)
class AuthInterceptor extends Interceptor {
  final TokenStorage _storage;

  const AuthInterceptor(this._storage);

  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) {
    final token = _storage.accessToken;
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    handler.next(options); // пропускаем запрос дальше
  }

  @override
  void onError(DioException err, ErrorInterceptorHandler handler) async {
    // Автоматически обновляем токен при 401
    if (err.response?.statusCode == 401) {
      try {
        await _storage.refreshToken();
        // Повторяем оригинальный запрос с новым токеном
        final response = await _retry(err.requestOptions);
        handler.resolve(response);
        return;
      } catch (_) {
        // refresh тоже упал — разлогиниваем
        _storage.clearTokens();
      }
    }
    handler.next(err);
  }

  Future<Response<dynamic>> _retry(RequestOptions requestOptions) {
    return Dio().fetch(requestOptions);
  }
}

// Логирующий интерцептор (только для debug)
import 'package:dio/dio.dart';

dio.interceptors.add(
  LogInterceptor(
    requestBody: true,
    responseBody: true,
    error: true,
  ),
);

// Добавление интерцепторов
dio.interceptors.addAll([
  AuthInterceptor(tokenStorage),
  LogInterceptor(requestBody: true),
]);
```

## 5. Реальный пример — API клиент с авторизацией

```dart
class ApiClient {
  late final Dio _dio;

  ApiClient(String baseUrl, TokenStorage tokenStorage) {
    _dio = Dio(
      BaseOptions(
        baseUrl: baseUrl,
        connectTimeout: const Duration(seconds: 10),
        receiveTimeout: const Duration(seconds: 30),
        headers: const {'Accept': 'application/json'},
      ),
    );

    _dio.interceptors.addAll([
      AuthInterceptor(tokenStorage),
      if (kDebugMode) LogInterceptor(requestBody: true, responseBody: true),
    ]);
  }

  Future<T> get<T>(String path, {Map<String, dynamic>? params}) async {
    final response = await _dio.get<T>(path, queryParameters: params);
    return response.data as T;
  }

  Future<T> post<T>(String path, {Object? data}) async {
    final response = await _dio.post<T>(path, data: data);
    return response.data as T;
  }

  Future<void> uploadFile(
    String path,
    File file, {
    ProgressCallback? onProgress,
  }) async {
    final formData = FormData.fromMap({
      'file': await MultipartFile.fromFile(
        file.path,
        filename: file.path.split('/').last,
      ),
    });

    await _dio.post(
      path,
      data: formData,
      onSendProgress: onProgress,
    );
  }
}

// --- Repository ---
class UserRepository {
  final ApiClient _client;

  const UserRepository(this._client);

  Future<User> getCurrentUser() async {
    final data = await _client.get<Map<String, dynamic>>('/me');
    return User.fromJson(data);
  }

  Future<List<User>> getUsers({int page = 1}) async {
    final data = await _client.get<List<dynamic>>(
      '/users',
      params: {'page': page},
    );
    return data.map((json) => User.fromJson(json as Map<String, dynamic>)).toList();
  }
}
```

## 6. Отмена запросов (CancelToken)

```dart
class SearchRepository {
  final Dio _dio;
  CancelToken? _searchToken;

  SearchRepository(this._dio);

  Future<List<Product>> search(String query) async {
    // Отменяем предыдущий запрос поиска
    _searchToken?.cancel('Новый поиск');
    _searchToken = CancelToken();

    try {
      final response = await _dio.get<List<dynamic>>(
        '/products/search',
        queryParameters: {'q': query},
        cancelToken: _searchToken,
      );
      return response.data!
          .map((json) => Product.fromJson(json as Map<String, dynamic>))
          .toList();
    } on DioException catch (e) {
      if (CancelToken.isCancel(e)) return []; // Запрос был отменён — не ошибка
      rethrow;
    }
  }
}
```

## 7. Под капотом

- `Dio` использует `HttpClient` из `dart:io` под капотом.
- `Interceptor` — цепочка обработчиков (аналог middleware): `onRequest → onResponse → onError`.
- `FormData` сериализуется в `multipart/form-data` автоматически.
- `CancelToken` основан на `dart:async` `Completer` — не реальная отмена TCP-соединения, а выброс исключения при первой возможности.

---

## 8. Типичные ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `DioExceptionType.connectionTimeout` | Сервер не отвечает | Проверь URL и сеть |
| `DioExceptionType.badResponse` | statusCode ≥ 400 | Обработай в `onError` интерцептора |
| `type 'List<dynamic>' is not a subtype of 'Map'` | Неверный generic тип | Укажи правильный тип `dio.get<List<dynamic>>` |
| Токен обновляется параллельно несколькими запросами | Race condition в refresh | Используй `Completer` для lock refresh |
| Логи в продакшне | `LogInterceptor` подключён всегда | Обворачивай в `if (kDebugMode)` |

---

## 9. Рекомендации

1. **Один экземпляр `Dio`** на всё приложение — создавай через DI.
2. **Интерцептор для токена** — не дублируй заголовок `Authorization` в каждом запросе.
3. **`CancelToken`** для поиска и любых запросов, которые могут быть отменены.
4. **Логирование только в debug** — `if (kDebugMode) LogInterceptor(...)`.
5. **Не лови `DioException` в виджетах** — выбрасывай доменные исключения из Repository.
6. **Типизируй generic** — `dio.get<Map<String, dynamic>>()` лучше `dio.get()`.
