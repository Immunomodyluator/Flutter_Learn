# 10.4 Firebase Cloud Messaging (FCM) — Push-уведомления

## 1. Суть

FCM — сервис Firebase для отправки push-уведомлений на Android, iOS и Web. Уведомления могут приходить в три состояния: foreground (приложение открыто), background (свёрнуто), terminated (закрыто).

```yaml
dependencies:
  firebase_messaging: ^14.7.19
  flutter_local_notifications: ^16.3.2 # для показа уведомлений в foreground
```

---

## 2. Настройка платформ

**Android** — `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>

<application ...>
  <!-- FCM default channel -->
  <meta-data
    android:name="com.google.firebase.messaging.default_notification_channel_id"
    android:value="high_importance_channel"/>
</application>
```

**iOS** — включить Push Notifications в XCode → Signing & Capabilities.

## 3. Инициализация и получение токена

```dart
import 'package:firebase_messaging/firebase_messaging.dart';

// Обработчик background-сообщений (top-level функция!)
@pragma('vm:entry-point')
Future<void> _firebaseMessagingBackgroundHandler(RemoteMessage message) async {
  await Firebase.initializeApp();
  print('Background message: ${message.messageId}');
}

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);

  // Регистрируем background handler
  FirebaseMessaging.onBackgroundMessage(_firebaseMessagingBackgroundHandler);

  runApp(const MyApp());
}

// --- В initState или при запуске ---
class NotificationService {
  final FirebaseMessaging _messaging = FirebaseMessaging.instance;

  Future<void> init() async {
    // Запрос разрешения (iOS обязателен, Android 13+)
    final settings = await _messaging.requestPermission(
      alert: true,
      badge: true,
      sound: true,
    );

    if (settings.authorizationStatus == AuthorizationStatus.authorized) {
      await _setupHandlers();
      await _getToken();
    }
  }

  Future<void> _getToken() async {
    final token = await _messaging.getToken();
    if (token != null) {
      print('FCM Token: $token');
      // Отправь токен на свой сервер для сохранения
      await _saveTokenToServer(token);
    }

    // Обновление токена (iOS после reinstall, Android после app data clear)
    _messaging.onTokenRefresh.listen(_saveTokenToServer);
  }

  Future<void> _setupHandlers() async {
    // Foreground — приложение открыто
    FirebaseMessaging.onMessage.listen(_handleForegroundMessage);

    // Background/Terminated — пользователь нажал на уведомление
    FirebaseMessaging.onMessageOpenedApp.listen(_handleNotificationTap);

    // Приложение было открыто через уведомление после terminated состояния
    final initial = await _messaging.getInitialMessage();
    if (initial != null) _handleNotificationTap(initial);
  }

  void _handleForegroundMessage(RemoteMessage message) {
    // В foreground Firebase НЕ показывает уведомление автоматически
    // Нужно показать через flutter_local_notifications
    _showLocalNotification(message);
  }

  void _handleNotificationTap(RemoteMessage message) {
    final data = message.data;
    // Навигация по данным из уведомления
    final route = data['route'] as String?;
    if (route != null) {

    }
  }

  Future<void> _saveTokenToServer(String token) async {

  void _showLocalNotification(RemoteMessage message) {
```

## 4. Показ уведомлений в foreground (flutter_local_notifications)

```dart
import 'package:flutter_local_notifications/flutter_local_notifications.dart';

class LocalNotificationsService {
  static final _plugin = FlutterLocalNotificationsPlugin();

  static Future<void> init() async {
    const androidSettings = AndroidInitializationSettings('@mipmap/ic_launcher');
    const iosSettings = DarwinInitializationSettings();

    await _plugin.initialize(
      const InitializationSettings(android: androidSettings, iOS: iosSettings),
    );

    // Создать канал (Android 8+)
    await _plugin
        .resolvePlatformSpecificImplementation<
            AndroidFlutterLocalNotificationsPlugin>()
        ?.createNotificationChannel(
          const AndroidNotificationChannel(
            'high_importance_channel',
            'Важные уведомления',
            importance: Importance.max,
          ),
        );
  }

  static Future<void> show(RemoteMessage message) async {
    final notification = message.notification;
    if (notification == null) return;

    await _plugin.show(
      notification.hashCode,
      notification.title,
      notification.body,
      const NotificationDetails(
        android: AndroidNotificationDetails(
          'high_importance_channel',
          'Важные уведомления',
          importance: Importance.max,
          priority: Priority.high,
        ),
      ),
      payload: message.data.toString(),
    );
  }
}
```

## 6. Под капотом

- FCM token — уникальный идентификатор устройство+приложение для адресации уведомлений.
- Background handler — запускается в отдельном Isolate, поэтому нужна отдельная инициализация Firebase.
- iOS требует Apple Push Notification Service (APNs) — нужны сертификаты в Firebase Console.
- `onMessage` — только foreground, `onMessageOpenedApp` — tap по background-уведомлению.

---

## 7. Типичные ошибки

| Ошибка                           | Причина                                         | Решение                                                             |
| -------------------------------- | ----------------------------------------------- | ------------------------------------------------------------------- |
| Уведомления не приходят на iOS   | APNs не настроен                                | Добавь APNs key в Firebase Console → Project Settings               |
| Background handler не вызывается | Функция не `top-level` или без `@pragma`        | Убедись что функция вне классов, добавь `@pragma('vm:entry-point')` |
| Пустой foreground                | Firebase не показывает уведомления в foreground | Используй `flutter_local_notifications` в `onMessage`               |
| Токен не обновляется             | Не подписан на `onTokenRefresh`                 | Всегда слушай `onTokenRefresh` и обновляй на сервере                |
| Не работает навигация при тапе   | Нет `getInitialMessage()`                       | Обрабатывай `getInitialMessage()` при старте приложения             |

---

## 8. Рекомендации

1. **Всегда обрабатывай все три состояния**: foreground (`onMessage`), background/tap (`onMessageOpenedApp`), terminated (`getInitialMessage`).
2. **Храни FCM токен на своём сервере** — обновляй при `onTokenRefresh`.
3. **`flutter_local_notifications`** для foreground уведомлений — Firebase не показывает их сам.
4. **Топики** для массовых рассылок, **targeted** (по токену) для персональных.
5. **Не храни чувствительные данные в payload** — уведомления видны в шторке.
