# 3. Основы виджетов Flutter

## 1. Суть концепции

**Всё во Flutter — виджет.** Виджет — это неизменяемое описание части UI (конфигурация, а не сам отображаемый элемент). Flutter строит дерево виджетов, по которому создаёт дерево Element, а затем дерево RenderObject.

```
Widget Tree        Element Tree       RenderObject Tree
(конфигурация)  →  (состояние)    →   (рисование)
```

---

## 2. Три дерева Flutter

| Дерево       | Роль                                            | Пересоздаётся                  |
| ------------ | ----------------------------------------------- | ------------------------------ |
| Widget       | Конфигурация UI                                 | На каждый rebuild (легковесно) |
| Element      | Связь виджета с состоянием, хранит BuildContext | Переиспользуется при rebuild   |
| RenderObject | Layout + Paint (тяжёлый)                        | Только при смене типа виджета  |

Это объясняет почему Flutter не медленный несмотря на постоянное пересоздание виджетов — пересоздаётся только лёгкая конфигурация.

---

## 3. Иерархия базовых виджетов

```dart
// Каждое Flutter-приложение — дерево виджетов
void main() {
  runApp(
    MaterialApp(               // корневой виджет
      home: Scaffold(          // базовый экран
        appBar: AppBar(
          title: const Text('Hello'),
        ),
        body: const Center(
          child: Text('World'),
        ),
      ),
    ),
  );
}
```

---

## 4. Типы виджетов

### StatelessWidget

Неизменяемый, без состояния. Перестраивается только если изменился родитель.

### StatefulWidget

Имеет изменяемый `State`. Может вызывать `setState()` для перерисовки.

### InheritedWidget

Передаёт данные вниз по дереву без явной передачи через параметры.

### RenderObjectWidget

Низкоуровневый виджет, напрямую управляет RenderObject (Column, Row, Container).

---

## 5. Жизненный цикл виджета

```
Widget создан → createElement() → mount() → build() → ...rebuild... → unmount()
```

Для StatefulWidget:

```
createState() → initState() → build() → setState() → build() → dispose()
```

## 6. Ключи (Keys) — идентификация виджетов

```dart
// Без ключа Flutter идентифицирует виджет по типу и позиции
// С ключом — по конкретному идентификатору

ListView(
  children: items.map((item) =>
    ItemWidget(key: ValueKey(item.id), item: item)
  ).toList(),
)
```

| Тип ключа   | Когда использовать                 |
| ----------- | ---------------------------------- |
| `ValueKey`  | Уникальное значение (id, строка)   |
| `ObjectKey` | Уникальный объект                  |
| `UniqueKey` | Каждый раз новый (сброс состояния) |
| `GlobalKey` | Доступ к `State` из любого места   |

## 7. BuildContext

`BuildContext` — ссылка на позицию виджета в дереве Element. Используется для:

- `Theme.of(context)` — получить тему
- `Navigator.of(context)` — навигация
- `MediaQuery.of(context)` — размер экрана
- `Provider.of(context)` — получить зависимость

```dart
@override
Widget build(BuildContext context) {
  final theme = Theme.of(context);         // ищет вверх по дереву
  final size = MediaQuery.of(context).size;
  return Text('Width: ${size.width}', style: theme.textTheme.bodyLarge);
}
```

## 8. Типичные ошибки

| Ошибка                         | Причина                           | Правильно                            |
| ------------------------------ | --------------------------------- | ------------------------------------ |
| Слишком большие build() методы | Один огромный метод               | Разбивать на отдельные виджеты       |
| Не используется `const`        | Лишние rebuilds                   | `const Text('Hello')`                |
| `GlobalKey` везде              | Антипаттерн, нарушает архитектуру | Использовать только по необходимости |
| `context` после async          | `context` может устареть          | Проверять `mounted`                  |

---

## 9. Практические рекомендации

1. **Виджет = маленькая, переиспользуемая единица** — если `build()` больше 50 строк, разбей.
2. **Используй `const` везде где возможно** — `const` виджеты не пересоздаются.
3. **Не передавай `BuildContext` в сервисы** — только в `build()` и коллбэках виджетов.
4. **Предпочитай composition над inheritance** — оборачивай виджеты, а не наследуй.
5. **Именуй виджеты по смыслу**, а не по внешнему виду: `UserAvatarWidget`, а не `RoundImageWidget`.
6. **Один файл = один публичный виджет** — вспомогательные приватные виджеты в том же файле.
