# 1. Введение в Dart

## 1. Суть концепции

Dart — строго типизированный, объектно-ориентированный язык, разработанный Google специально как основа для Flutter. Компилируется в нативный машинный код (AOT) для продакшена и работает через JIT в режиме разработки (что и обеспечивает Hot Reload).

Ключевые факты:

- **Синтаксис** близок к Java/Kotlin/C# — низкий порог входа для backend/mobile разработчиков
- **Null Safety** встроен в систему типов — класс ошибок NPE устранён на уровне компилятора
- **Single-threaded** с асинхронной моделью на основе Event Loop (как JavaScript, но с изоляторами для параллелизма)
- **Всё — объект**, включая функции и `null`

---

## 2. Место Dart в экосистеме Flutter

| Слой       | Технология                                  |
| ---------- | ------------------------------------------- |
| UI         | Flutter widgets (Dart)                      |
| Логика     | Dart классы, функции, async                 |
| Платформа  | Dart ↔ Kotlin/Swift через Platform Channels |
| Компиляция | JIT (dev) → AOT (release)                   |

Flutter не использует WebView и не транслирует код в нативные компоненты. Dart компилируется напрямую, и Flutter сам рисует каждый пиксель через Skia/Impeller.

---

## 3. JIT vs AOT — почему это важно

```
Разработка (flutter run):
  Dart VM → JIT компиляция → Hot Reload работает

Продакшен (flutter build apk/ipa):
  dart2native → AOT → бинарный код → нет VM overhead
```

**JIT** (Just-In-Time): код компилируется во время исполнения. Позволяет Dart VM менять код на лету → это и есть механика Hot Reload.

**AOT** (Ahead-Of-Time): весь код компилируется заранее в машинный код. Быстрый старт, нет overhead VM, но нет динамической подстановки кода.

## 4. Минимальный Dart-файл

```dart
void main() {
  print('Hello, Flutter!');
}
```

- `main()` — точка входа любого Dart-приложения
- `void` — функция ничего не возвращает
- Точка с запятой обязательна

В Flutter `main()` выглядит так:

```dart
void main() {
  runApp(const MyApp()); // запускает дерево виджетов
}
```

## 5. Что происходит под капотом

### Event Loop

Dart однопоточный. Все асинхронные операции (сеть, таймеры, I/O) обрабатываются через Event Loop:

```
main() → выполнение синхронного кода
       → microtask queue (Future.value, scheduleMicrotask)
       → event queue (таймеры, I/O, UI events)
       → повтор цикла
```

Это значит: долгая синхронная операция (цикл на 1 млн итераций) **заморозит UI**, потому что Dart не переключится на рендеринг, пока не закончит синхронный код.

### Isolates

Для параллельного выполнения тяжёлых задач Dart использует `Isolate` — отдельный поток с собственной памятью и Event Loop. Общаются изоляторы только через сообщения (нет общей памяти → нет race conditions).

```dart
// В Flutter для простых случаев используют compute():
final result = await compute(heavyFunction, inputData);
```

## 6. Типичные ошибки

| Ошибка                                | Причина                                         | Правильно                                                    |
| ------------------------------------- | ----------------------------------------------- | ------------------------------------------------------------ |
| Выполнение тяжёлой работы в `build()` | `build()` вызывается часто, блокирует UI thread | Вычисления в `initState()`, `FutureBuilder`, или `compute()` |
| Использование `dynamic` везде         | Теряется null safety и подсказки IDE            | Указывать конкретные типы                                    |
| Игнорирование `await`                 | Future выполняется, но результат не ожидается   | Всегда `await` там, где нужен результат                      |
| `print()` в продакшен-коде            | Замедляет приложение, засоряет логи             | Использовать `debugPrint()` или пакет `logger`               |

## 7. Практические рекомендации

1. **Используй `dart analyze`** перед каждым коммитом — статический анализ поймает большинство ошибок до запуска.
2. **Форматируй код через `dart format .`** — в Dart есть официальный стиль, не изобретай свой.
3. **`const` везде где возможно** — компилятор создаёт объект один раз, Flutter не перерисовывает `const` виджеты без необходимости.
4. **Не злоупотребляй `dynamic`** — теряешь null safety, автодополнение и рефакторинг.
5. **Dart — не JavaScript**: несмотря на похожую async-модель, Dart строго типизирован и компилируется. Подходы "утиной типизации" здесь не работают.
6. **Изучи DartPad** (dartpad.dev) — онлайн-песочница для экспериментов с Dart без установки Flutter.
7. **`late` с осторожностью** — позволяет отложить инициализацию, но вызовет `LateInitializationError` если обратиться до инициализации.
