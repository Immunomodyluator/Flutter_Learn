# 2.2 Flutter CLI

## 1. Суть концепции

Flutter CLI — командная строка для управления проектом: создание, запуск, сборка, тестирование, управление зависимостями. Знание основных команд обязательно для эффективной разработки.

---

## 2. Основные команды

### Создание проекта

```bash
flutter create my_app                          # создать проект
flutter create --org com.company my_app        # с org (для app store)
flutter create --template=package my_package   # создать пакет
flutter create --template=plugin my_plugin     # создать плагин
```

### Запуск

```bash
flutter run                    # запустить на подключённом устройстве
flutter run -d chrome          # в браузере
flutter run -d windows         # на Windows Desktop
flutter run --release          # в release режиме (без debug overlay)
flutter run --profile          # profile режим (для профилирования)
flutter run --dart-define=API_URL=https://api.example.com  # переменные окружения
```

### Устройства

```bash
flutter devices                # список подключённых устройств
flutter emulators              # список доступных эмуляторов
flutter emulators --launch Pixel_6_API_33  # запустить эмулятор
```

### Зависимости

```bash
flutter pub get                # установить зависимости из pubspec.yaml
flutter pub upgrade            # обновить зависимости до последних версий
flutter pub add http           # добавить пакет
flutter pub add --dev mockito  # добавить dev-зависимость
flutter pub remove http        # удалить пакет
flutter pub outdated           # показать устаревшие пакеты
flutter pub publish            # опубликовать пакет на pub.dev
```

### Сборка

```bash
# Android
flutter build apk              # APK (все ABI)
flutter build apk --split-per-abi  # отдельные APK по архитектуре (меньше размер)
flutter build appbundle        # AAB для Google Play (рекомендуется)
flutter build apk --release    # release сборка (с обфускацией и оптимизацией)

# iOS
flutter build ios              # сборка для iOS
flutter build ipa              # .ipa для App Store

# Web
flutter build web              # production сборка
flutter build web --web-renderer canvaskit  # выбор renderer

# Desktop
flutter build windows
flutter build macos
flutter build linux
```

### Анализ и форматирование

```bash
flutter analyze                # статический анализ кода (dart analyze)
flutter format .               # форматирование кода (dart format .)
flutter test                   # запустить все тесты
flutter test test/unit/        # тесты в конкретной папке
flutter test --coverage        # с генерацией coverage report
```

### Очистка

```bash
flutter clean                  # удалить build/ и .dart_tool/
flutter pub cache clean        # очистить кеш pub
```

### Диагностика

```bash
flutter doctor                 # диагностика окружения
flutter doctor -v              # подробный вывод
flutter --version              # текущая версия Flutter
flutter upgrade                # обновить Flutter до последней stable
flutter channel                # показать доступные каналы
flutter channel stable         # переключиться на канал
```

## 3. Флаги сборки

```bash
# Dart define — передача переменных окружения в код
flutter run --dart-define=ENV=development
flutter run --dart-define=API_URL=https://dev.api.com --dart-define=DEBUG=true

# В коде читается так:
const env = String.fromEnvironment('ENV', defaultValue: 'production');
const apiUrl = String.fromEnvironment('API_URL');
```

---

## 4. Практические рекомендации

1. **`flutter pub get` вместо `pub get`** — используй Flutter-обёртку для правильной привязки к версии Flutter.
2. **`flutter build appbundle`** вместо `apk` для Google Play — меньший размер загрузки для пользователей.
3. **`flutter clean`** при странных ошибках компиляции — решает 80% "магических" проблем.
4. **`flutter analyze` в CI** — обязательный шаг перед merge.
5. **`--dart-define`** для разных окружений (dev/staging/prod) вместо хардкода URL в коде.
