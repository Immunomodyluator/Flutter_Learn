# 17.1 Flutter Web

## 1. Суть

Flutter Web компилирует Dart-код в JavaScript (или WebAssembly). Рендерит через canvas или HTML DOM. Хорош для SPA-приложений, дашбордов, внутренних инструментов.

---

## 2. Рендереры

| Renderer                | Плюсы                                   | Минусы                            |
| ----------------------- | --------------------------------------- | --------------------------------- |
| **CanvasKit** (default) | Точный рендеринг = мобильное приложение | Размер ~1.5 МБ, нет SEO           |
| **HTML**                | Маленький размер, SEO-доступен          | Менее точный рендеринг            |
| **WebAssembly**         | Лучшая производительность               | Требует Dart 3.3+, большой размер |

```bash
# Выбор рендерера при запуске
flutter run -d chrome --web-renderer html
flutter run -d chrome --web-renderer canvaskit

# При сборке
flutter build web --web-renderer html
flutter build web --web-renderer canvaskit     # по умолчанию
flutter build web --wasm                        # WebAssembly (Dart 3.3+)
```

## 3. Сборка и деплой

```bash
# Сборка в build/web/
flutter build web --release --base-href /myapp/

# base-href если не в корне домена
flutter build web --release --base-href /
```

**Hosting варианты:**

```bash
# Firebase Hosting
firebase init hosting
flutter build web && firebase deploy

# GitHub Pages
flutter build web --base-href /repo-name/
# Скопировать build/web/* в gh-pages ветку

# Vercel / Netlify
# Build command: flutter build web
# Output directory: build/web
# Install command: (установить Flutter в pipeline)
```

## 4. index.html — настройка

```html
<!-- web/index.html -->
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Flutter App</title>

    <!-- Иконка -->
    <link rel="icon" type="image/png" href="icons/Icon-192.png" />

    <!-- PWA Manifest -->
    <link rel="manifest" href="manifest.json" />

    <!-- Загрузочный экран вместо белого фона -->
    <style>
      body {
        background: #1a1a2e;
      }
      .loading {
        display: flex;
        justify-content: center;
        align-items: center;
        height: 100vh;
      }
    </style>
  </head>
  <body>
    <div id="loading" class="loading">
      <img src="icons/Icon-192.png" width="64" />
    </div>

    <script src="flutter_bootstrap.js" async></script>
    <script>
      window.addEventListener("flutter-first-frame", () => {
        document.getElementById("loading").remove();
      });
    </script>
  </body>
</html>
```

## 5. PWA (Progressive Web App)

Flutter Web генерирует `manifest.json` и service worker автоматически.

```json
// web/manifest.json
{
  "name": "My Flutter App",
  "short_name": "MyApp",
  "start_url": ".",
  "display": "standalone",
  "background_color": "#1a1a2e",
  "theme_color": "#6366f1",
  "description": "Описание приложения",
  "orientation": "portrait-primary",
  "prefer_related_applications": false,
  "icons": [
    { "src": "icons/Icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "icons/Icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

## 6. URL стратегии

```dart
// main.dart
import 'package:flutter_web_plugins/url_strategy.dart';

void main() {
  usePathUrlStrategy(); // /home вместо /#/home
  runApp(const MyApp());
}
```

```nginx
# nginx — перенаправлять все URL на index.html (SPA routing)
server {
    listen 80;
    root /var/www/flutter_app;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 7. Web-совместимость кода

```dart
// Проверка платформы для web-специфичного кода
import 'package:flutter/foundation.dart';

Widget buildLink(String url, String text) {
  if (kIsWeb) {
    return HtmlElementView(
      viewType: 'link-element',
      // или используй url_launcher
    );
  }
  return TextButton(
    onPressed: () => launchUrl(Uri.parse(url)),
    child: Text(text),
  );
}

// Пакеты, не совместимые с Web
// - path_provider (частично)
// - sqflite (нет)
// - camera (ограниченно)
// Замены:
// - shared_preferences (работает в Web через localStorage)
// - idb_shim (IndexedDB для Web)
```

## 10. SEO

Flutter Web с CanvasKit плохо индексируется поисковиками (контент в canvas).

**Варианты:**

1. **HTML renderer** — лучше для SEO, но менее точный.
2. **Meta tags** в `index.html` — описание, заголовок.
3. **Pre-rendering** через `flutter_seo` пакет.
4. **SSR** — пока не поддерживается нативно Flutter.

```html
<!-- web/index.html -->
<meta name="description" content="Описание приложения для поисковиков" />
<meta property="og:title" content="My App" />
<meta property="og:description" content="Описание" />
<meta property="og:image" content="https://myapp.com/preview.png" />
```

## 11. Типичные ошибки

| Ошибка                 | Причина                         | Решение                             |
| ---------------------- | ------------------------------- | ----------------------------------- |
| CORS ошибки при fetch  | Сервер не разрешает web-запросы | Добавить CORS-заголовки на backend  |
| 404 при навигации      | SPA routing не настроен         | `usePathUrlStrategy` + nginx config |
| Пустая страница онлайн | Неверный `base-href`            | Указать правильный путь             |
| Шрифты не загружаются  | Нет интернета (PWA offline)     | Включить в service worker cache     |

---

## 12. Рекомендации

1. **CanvasKit** — для приложений (инструменты, дашборды). **HTML renderer** — для SEO.
2. **PWA** — настрой `manifest.json` — пользователи смогут «установить» приложение.
3. **`usePathUrlStrategy()`** — чистые URL без `#`.
4. **Lazy loading** через deferred imports — уменьши начальный бандл.
5. **CORS** на сервере — необходим для любых API-запросов с Web.
