# 11. Анимации во Flutter

## 1. Типы анимаций

Flutter предоставляет два уровня анимации:

| Тип                    | Виджеты                                                         | Когда использовать                        |
| ---------------------- | --------------------------------------------------------------- | ----------------------------------------- |
| **Implicit** (неявные) | `AnimatedContainer`, `AnimatedOpacity`, `TweenAnimationBuilder` | Простые переходы свойств                  |
| **Explicit** (явные)   | `AnimationController`, `Tween`, `AnimatedBuilder`               | Полный контроль, сложные анимации         |
| **Hero**               | `Hero`                                                          | Общий элемент при переходе между экранами |
| **Staggered**          | Явные + задержки                                                | Последовательные анимации                 |
| **Lottie/Rive**        | Сторонние пакеты                                                | Дизайнерские анимации                     |

---

## 2. Пирамида сложности

```
Lottie / Rive          ← готовые дизайнерские анимации
       ↑
CustomPainter + Animation ← пиксельный контроль
       ↑
AnimationController + Tween ← явные (полный контроль)
       ↑
TweenAnimationBuilder ← полуявные (проще)
       ↑
AnimatedFoo виджеты ← неявные (самые простые)
```

---

## 3. Принцип работы

1. **Ticker** — метроном анимации, срабатывает каждый кадр (~16ms при 60fps).
2. **AnimationController** — управляет значением от 0.0 до 1.0 во времени.
3. **Tween** — маппинг значений контроллера в нужный диапазон (например, `0.0 → 300.0` для ширины).
4. **AnimatedBuilder** — перестраивает виджет при каждом изменении анимации.

---

## 4. Рекомендации

1. **Начинай с Implicit виджетов** — `AnimatedContainer`, `AnimatedOpacity` решают 80% задач.
2. **`TickerProviderStateMixin`** нужен для `AnimationController` в StatefulWidget.
3. **Dispose контроллер** в `dispose()` — иначе утечка памяти.
4. **`const` ключевые части** — не анимируй то, что не меняется.
5. **60fps правило**: анимации должны работать без пропуска кадров. Проверяй в DevTools.
