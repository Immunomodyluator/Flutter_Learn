# 2.3 Hot Reload и Hot Restart

## 1. Суть концепции

Hot Reload и Hot Restart — инструменты быстрой итерации в режиме разработки. Работают только при `flutter run` (JIT компиляция). В release сборке недоступны.

---

## 2. Hot Reload (`r` в терминале или `⌘R` / `Ctrl+F5` в IDE)

**Что делает**: вливает обновлённый код в работающую Dart VM и пересобирает дерево виджетов — **без перезапуска приложения**.

**Что сохраняется**:

- Состояние приложения (`State` объекты)
- Позиция скролла
- Введённый текст
- Навигационный стек

**Что НЕ применяется**:

- Изменения в `initState()`, `dispose()`
- Изменения конструкторов виджетов
- Изменения в `main()`
- Новые глобальные переменные
- Изменения native кода (Android/iOS)

```
Время: ~300ms — практически мгновенно
```

---

## 3. Hot Restart (`R` в терминале или `⇧⌘R` / `Shift+F5` в IDE)

**Что делает**: полностью перезапускает Dart приложение — без пересборки нативной части.

**Что сбрасывается**:

- Всё состояние приложения
- Навигационный стек
- Все переменные → начальные значения

**Когда нужен**:

- После изменений в `initState()`
- После добавления нового глобального состояния
- Когда Hot Reload не применил изменения

```
Время: ~2-5 сек
```

---

## 4. Полный перезапуск (Stop + Run)

Пересобирает нативную часть. Нужен при:

- Изменении `pubspec.yaml` (новые зависимости)
- Изменениях в `android/` или `ios/`
- Первом запуске после `flutter clean`

```
Время: ~30-60 сек
```

---

## 5. Сравнение

|             | Hot Reload       | Hot Restart    | Full Restart   |
| ----------- | ---------------- | -------------- | -------------- |
| Состояние   | ✓ сохраняется    | ✗ сбрасывается | ✗ сбрасывается |
| Время       | ~300ms           | ~3 сек         | ~30-60 сек     |
| initState() | ✗ не обновляется | ✓              | ✓              |
| main()      | ✗                | ✓              | ✓              |
| Native код  | ✗                | ✗              | ✓              |

---

## 6. Практические рекомендации

1. **Hot Reload для UI-правок** — цвет, отступы, текст, расположение виджетов.
2. **Hot Restart после изменения логики `State`** — если что-то "не обновилось", сначала Hot Restart.
3. **Сохранение файла = автоматический Hot Reload** в VS Code при настройке `"dart.flutterHotReloadOnSave": "always"`.
4. **Если Hot Reload показывает устаревший UI** — сделай Hot Restart, это нормально.
5. **При изменении плагина** (добавление в pubspec.yaml + `flutter pub get`) — всегда полный перезапуск.
